EXPLORATION SUMMARY: Metadata Loss in Save Agent Graph
════════════════════════════════════════════════════════════════════════════════

PROBLEM STATEMENT:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
When users click the global "Save" button (top-right), metadata (permissions, 
skills, aspects) of agents is lost from .adata files, particularly the 
orchestrator. This happens after:
  1. User defines permissions in the Permissions Modal (separate IPC call ✅)
  2. User edits agent properties (name, type, etc.) in the canvas
  3. User clicks "Save" global button → permissions are gone ❌

════════════════════════════════════════════════════════════════════════════════

ROOT CAUSE: Code Path Analysis
════════════════════════════════════════════════════════════════════════════════

Flow: [Save Button] → [AgentGraphSaveButton.tsx] → [IPC: SAVE_AGENT_GRAPH] → 
      [ipc-handlers.ts] → [.adata file written]

STEP 1: Serialization Gap (AgentGraphSaveButton.tsx:78-87)
  Current: Serializes only 8 fields
    - id, name, description, type, isOrchestrator, hidden, x, y
  Missing: permissions, skills, aspects, metadata
  Status: ❌ INCOMPLETE

STEP 2: IPC Handler (ipc-handlers.ts:613-793)
  Good: Lines 716-724 read existing .adata from disk
  Good: Lines 731-740 preserve aspects, skills, subagents, profile
  Bad: Line 741-750 preserves metadata but SKIPS permissions
  Missing: No line `permissions: (existing.permissions as Record<string, unknown>) ?? undefined,`

STEP 3: Disk Write (ipc-handlers.ts:752)
  Result: .adata file written WITHOUT permissions field
  Status: ❌ CRITICAL

IMPACT ASSESSMENT
════════════════════════════════════════════════════════════════════════════════

Severity: 🔴 CRITICAL
  - Affects most critical component (orchestrator)
  - Permissions control what agents can do (Bash, file access, etc.)
  - User has no warning that permissions were lost
  - Data is silently overwritten

Likelihood: 🟡 MEDIUM
  - Only happens if:
    1. User defines permissions (not always done)
    2. User edits agent after that
    3. User clicks Save (happens frequently)
  - Three-step scenario but common workflow

Scope: HIGH
  - Affects ANY agent with permissions defined
  - Especially dangerous for orchestrator
  - Skills and aspects may also be at risk (not tested)
